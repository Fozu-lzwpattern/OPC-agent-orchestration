# 成本追踪方法

## 追踪维度

1. **按角色**：每个 Sub-agent 消耗多少
2. **按任务**：每个 Task 消耗多少
3. **按阶段**：策划/执行/发布各消耗多少
4. **总计**：项目总成本

## Sub-agent 汇报规范

每次 announce 必须包含：
```
### Token 消耗
约 {N}K tokens
```

## CEO 汇总

项目结束时生成成本报告：

```
📊 项目成本报告
项目：{project_name}
周期：{start} - {end}

| 角色 | 任务数 | Token | 成本估算 |
|------|--------|-------|---------|
| 策划员 | 2 | 35K | $0.09 |
| 搭建员 | 3 | 95K | $0.24 |
| 发布员 | 2 | 20K | $0.05 |
| CEO协调 | - | 30K | $0.08 |
| **总计** | **7** | **180K** | **$0.46** |

（按 $2.5/1M input tokens 估算）
```

## 预算告警

- 单任务超预算 50%：Sub-agent 应主动汇报
- 项目累计超预算 80%：CEO 告警用户
- 项目累计超预算 100%：暂停执行，等待指示

## 成本优化建议

1. **先小后大**：先用简单方案验证，再投入大预算
2. **复用产出**：Sub-agent 产出保存为文件，后续可直接读取
3. **裁剪角色**：简单项目合并角色，减少协调开销
